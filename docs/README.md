# Documentation

This directory contains project documentation.

## Structure

- `api/` — Generated by Doxygen (GITIGNORED)
- `design/` — Design documents and ADRs
  - `architecture.md` — Architecture overview
  - `adr/` — Architecture Decision Records
- `false-positives.md` — Documented static analysis suppressions

## Generating API Documentation

```bash
doxygen Doxyfile
# Output: docs/api/html/
```

## Adding Architecture Decisions

Create a new ADR in `docs/design/adr/`:

- Format: `ADR-NNN-<slug>.md`
- Template:

  ```markdown
  # ADR-NNN: <Title>

  **Date:** YYYY-MM-DD  
  **Status:** proposed | approved | deprecated | superseded

  ## Context

  ...

  ## Decision

  ...

  ## Consequences

  ...
  ```

## False Positives

Document any false positives from clang-tidy or scan-build in `false-positives.md`:

```markdown
### Check Name: <check-name>

- **File:** src/module/module.c:123
- **Line:** 123
- **Reason:** <explanation why this is a false positive>
- **Suppression:** `// NOLINT(<check-name>): <reason>`
```
