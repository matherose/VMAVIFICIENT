# CI Rework Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the single 419-line `build.yml` with a professional split CI — `ci.yml` (push/PR) + `release.yml` (tags/dispatch) sharing a composite setup action — and pin all vendored dependency tags to immutable commit SHAs.

**Architecture:** A composite action (`.github/actions/setup-build`) holds the previously copy-pasted preamble (brew, Rust, cargo-c, caches, test media). `ci.yml` runs lint (format + actionlint) gating build-test (build + unit tests + clang-tidy + smoke encode, absorbing the old clang-tidy and encode-test jobs) and sanitizer. `release.yml` is a single tag-gated-publish job that doubles as a dispatch dry-run. CMakeLists dep pins become commit SHAs so the exact-key caches are genuinely immutable.

**Tech Stack:** GitHub Actions (macos-14 + ubuntu-latest), CMake ExternalProject, actionlint, Homebrew, Rust/cargo-c.

**Spec:** `docs/superpowers/specs/2026-07-11-ci-rework-design.md` — the "Preserved behaviors" section there is a hard keep-list.

### Task 1: Composite setup action

**Files:**
- Create: `.github/actions/setup-build/action.yml`

- [ ] **Step 1: Create the directory and file**

Write `.github/actions/setup-build/action.yml` with exactly this content:

```yaml
name: Setup build environment
description: >
  Shared macOS toolchain setup: Homebrew packages, Rust + cargo-c,
  dependency caches, and (optionally) the Big Buck Bunny test media.
  Single source of truth for the brew lists — they drifted when this
  preamble was copy-pasted across four jobs.

inputs:
  mode:
    description: >
      "system" = Homebrew shared deps (+ llvm for clang-tidy);
      "vendored" = minimal brew, deps built by ExternalProject.
    required: true
  deps-path:
    description: Path of the ExternalProject deps tree to cache (e.g. build/deps)
    required: true
  deps-cache-key:
    description: Exact cache key for the deps tree
    required: true
  test-media:
    description: Also fetch Big Buck Bunny and cut sample.mp4
    required: false
    default: 'false'

runs:
  using: composite
  steps:
    - name: Update Homebrew
      shell: bash
      run: brew update

    - name: Install brew packages (system mode)
      if: inputs.mode == 'system'
      shell: bash
      run: |
        brew install \
          ninja pkg-config nasm meson llvm \
          ffmpeg opus dovi_tool tesseract leptonica libvmaf \
          jpeg-turbo libpng libtiff cjson openssl@3

    - name: Install brew packages (vendored mode)
      if: inputs.mode == 'vendored'
      shell: bash
      run: brew install ninja pkg-config nasm meson ffmpeg

    # Pinned to a commit on the "stable" branch; the toolchain input
    # replaces what the mutable @stable ref used to select.
    - name: Install Rust toolchain
      uses: dtolnay/rust-toolchain@4be7066ada62dd38de10e7b70166bc74ed198c30 # stable branch, 2026-07-11
      with:
        toolchain: stable

    # Keyed on CMakeLists.txt (it holds the grav1synth rev) so the
    # registry snapshot refreshes on dep bumps; the old static key could
    # never refresh because actions/cache entries are immutable.
    - name: Cache cargo registry & git
      uses: actions/cache@0057852bfaa89a56745cba8c7296529d2fc39830 # v4
      with:
        path: |
          ~/.cargo/registry
          ~/.cargo/git
        key: cargo-macos-arm64-${{ hashFiles('CMakeLists.txt') }}
        restore-keys: cargo-macos-arm64-

    # Exact-key only (no restore-keys): restoring a stale deps tree makes
    # ExternalProject run git update in place, which can fail to unstash
    # local changes when a pinned ref moves.
    - name: Cache ExternalProject builds
      uses: actions/cache@0057852bfaa89a56745cba8c7296529d2fc39830 # v4
      with:
        path: ${{ inputs.deps-path }}
        key: ${{ inputs.deps-cache-key }}

    - name: Install cargo-c
      shell: bash
      run: cargo install cargo-c

    - name: Cache test media
      if: inputs.test-media == 'true'
      id: bbb-cache
      uses: actions/cache@0057852bfaa89a56745cba8c7296529d2fc39830 # v4
      with:
        path: bbb.mp4.zip
        key: bbb-sunflower-1080p-30fps

    - name: Download test media
      if: inputs.test-media == 'true' && steps.bbb-cache.outputs.cache-hit != 'true'
      shell: bash
      run: |
        curl -L -o bbb.mp4.zip \
          "https://download.blender.org/demo/movies/BBB/bbb_sunflower_1080p_30fps_normal.mp4.zip"

    - name: Prepare test media
      if: inputs.test-media == 'true'
      shell: bash
      run: |
        unzip -o bbb.mp4.zip
        ffmpeg -y -ss 0 -t 5 -i bbb_sunflower_1080p_30fps_normal.mp4 \
          -c copy sample.mp4
```

### Task 2: ci.yml

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Create the workflow**

Write `.github/workflows/ci.yml` with exactly this content:

```yaml
name: CI

on:
  push:
    branches: [main]
    paths-ignore: ['docs/**', '**.md']
  pull_request:
    branches: [main]
    paths-ignore: ['docs/**', '**.md']

# Superseded runs on the same ref are wasted macOS minutes.
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

permissions:
  contents: read

jobs:
  # ── Lint: format + workflow files ─────────────────────────────────
  # Cheap, runs on a tiny ubuntu image; fails fast on style drift so we
  # don't waste 30 minutes of macOS time before noticing.
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@93cb6efe18208431cddfb8368fd83d5badbf9bfd # v5

      # Pin to a specific clang-format version. The unversioned package
      # tracks the distro default and shifts under our feet when the
      # runner image updates; pinning keeps the gate stable so a green
      # local run stays green in CI.
      - name: Install clang-format
        run: sudo apt-get update && sudo apt-get install -y clang-format-18

      - name: Check formatting
        run: |
          clang-format-18 --dry-run -Werror \
            $(find src include -name "*.c" -o -name "*.h")

      # Download script referenced by commit (not tag) and version passed
      # explicitly, so neither can move underneath us.
      - name: Install actionlint
        run: |
          bash <(curl -sSfL "https://raw.githubusercontent.com/rhysd/actionlint/a443f344ff32813837fa49f7aa6cbc478d770e62/scripts/download-actionlint.bash") 1.7.9

      - name: Lint workflows
        run: ./actionlint -color

  # ── Build + test (system deps) ─────────────────────────────────────
  # Verifies the recommended build path (VMAV_USE_SYSTEM_DEPS=ON, the
  # default since v1.1.0). Absorbs the old clang-tidy job (reuses this
  # build's compile_commands.json instead of building twice) and the old
  # encode-test job (runs the installed binary in-place instead of
  # round-tripping an artifact).
  build-test:
    runs-on: macos-14
    timeout-minutes: 90
    needs: lint
    steps:
      - uses: actions/checkout@93cb6efe18208431cddfb8368fd83d5badbf9bfd # v5

      - uses: ./.github/actions/setup-build
        with:
          mode: system
          deps-path: build/deps
          deps-cache-key: cmake-deps-macos-arm64-${{ hashFiles('CMakeLists.txt') }}
          test-media: 'true'

      - name: Configure (system deps, compile commands)
        run: |
          cmake -G Ninja -B build \
            -DCMAKE_BUILD_TYPE=Release \
            -DCMAKE_EXPORT_COMPILE_COMMANDS=ON \
            -DOPENSSL_ROOT_DIR="$(brew --prefix openssl@3)" \
            -DVMAV_GRAV1SYNTH_BIN_RUNTIME=grav1synth

      - name: Build
        run: cmake --build build -j$(sysctl -n hw.ncpu)

      - name: Run unit tests
        run: ./build/vmav_tests

      - name: Verify version banner
        run: ./build/vmavificient --help | head -1

      # Zero-findings policy (.clang-tidy, WarningsAsErrors) enforced over
      # the whole tree — pre-commit only checks changed files.
      # run-clang-tidy resolves the clang-tidy binary via $PATH, not
      # relative to itself, and brew keeps llvm/bin off PATH — pass it
      # explicitly or the wrapper dies with "failed to find clang-tidy".
      - name: Run clang-tidy
        run: |
          LLVM="$(brew --prefix llvm)"
          "$LLVM/bin/run-clang-tidy" \
            -clang-tidy-binary "$LLVM/bin/clang-tidy" \
            -p build -quiet \
            $(find src -name '*.c')

      # Install to a prefix and run from there: verifies the packaging
      # path works — binary + grav1synth side-by-side in bin/.
      - name: Install (packaging check)
        run: |
          cmake --install build --prefix "$GITHUB_WORKSPACE/dist"
          chmod +x "$GITHUB_WORKSPACE/dist/bin/vmavificient"

      # Still a smoke test, not a quality gate.
      - name: Smoke encode (installed binary)
        timeout-minutes: 15
        run: |
          PATH="$GITHUB_WORKSPACE/dist/bin:$PATH" \
            dist/bin/vmavificient --blind --bitrate 2000 sample.mp4

      - name: Verify output
        run: |
          OUTPUT=$(ls -1 *.mkv 2>/dev/null | head -1)
          [ -n "$OUTPUT" ] || { echo "::error::no MKV produced"; exit 1; }
          echo "Output: $OUTPUT ($(du -h "$OUTPUT" | cut -f1))"

  # ── Sanitizer build (ASAN/UBSAN, vendored) ─────────────────────────
  sanitizer:
    runs-on: macos-14
    timeout-minutes: 120
    needs: lint
    steps:
      - uses: actions/checkout@93cb6efe18208431cddfb8368fd83d5badbf9bfd # v5

      - uses: ./.github/actions/setup-build
        with:
          mode: vendored
          deps-path: build-asan/deps
          deps-cache-key: cmake-deps-asan-macos-arm64-${{ hashFiles('CMakeLists.txt') }}
          test-media: 'true'

      - name: Configure (sanitizer + vendored)
        run: |
          cmake -G Ninja -B build-asan \
            -DCMAKE_BUILD_TYPE=Debug \
            -DVMAV_USE_SYSTEM_DEPS=OFF \
            -DVMAV_SANITIZE=ON

      - name: Build (sanitizer)
        run: cmake --build build-asan -j$(sysctl -n hw.ncpu)

      - name: Run unit tests (sanitizer)
        env:
          ASAN_OPTIONS: halt_on_error=1:detect_leaks=0
          UBSAN_OPTIONS: halt_on_error=1:print_stacktrace=1
        run: ./build-asan/vmav_tests

      # ASAN_OPTIONS=halt_on_error=1 so a single bad access fails CI loud.
      # detect_leaks=0 because libdovi/libhdr10plus's Rust runtime hangs
      # on tiny static allocations the loader doesn't reclaim — false
      # positives, not real leaks.
      - name: Test encode (sanitizer)
        env:
          ASAN_OPTIONS: halt_on_error=1:detect_leaks=0
          UBSAN_OPTIONS: halt_on_error=1:print_stacktrace=1
        run: ./build-asan/vmavificient --blind --bitrate 2000 sample.mp4
        timeout-minutes: 30

# Linux CI is deferred until the dep surface is in a stable distro most
# users actually run (libdovi-dev only landed in Debian trixie / Ubuntu
# 25.04+). When we revisit, mirror build-test on ubuntu-latest with
# apt-installed -dev packages.
```

- [ ] **Step 2: Create the actionlint config**

actionlint shellchecks every `run:` block. The unquoted command substitutions in these workflows (`$(find …)` file lists, `-j$(sysctl -n hw.ncpu)`) are intentional word-splitting, but shellcheck flags them as SC2046 and actionlint would fail the lint job on its own workflow. Write `.github/actionlint.yaml`:

```yaml
paths:
  .github/workflows/**/*.yml:
    ignore:
      # SC2046 (unquoted $(...)) — the word splitting is intentional in
      # every occurrence here: file lists fed to clang-format/clang-tidy
      # and -j$(sysctl -n hw.ncpu).
      - 'shellcheck reported issue in this script: SC2046:.*'
```

### Task 3: release.yml

**Files:**
- Create: `.github/workflows/release.yml`

- [ ] **Step 1: Create the workflow**

Write `.github/workflows/release.yml` with exactly this content:

```yaml
name: Release

on:
  push:
    tags: ['v*']
  # Dispatch = dry-run validator: the whole release path runs, only the
  # publish step is tag-gated. No throwaway tag needed.
  workflow_dispatch:

# Never cancel a release mid-publish.
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: false

permissions:
  contents: read

jobs:
  release:
    runs-on: macos-14
    timeout-minutes: 150
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@93cb6efe18208431cddfb8368fd83d5badbf9bfd # v5

      # Guard against tagging a commit whose project() version disagrees
      # with the tag — the CLI banner shipped three releases reporting
      # v1.2.0 because nothing checked this. Runs FIRST so a mismatched
      # tag fails in seconds, not after a two-hour vendored build.
      - name: Verify tag matches project version
        if: startsWith(github.ref, 'refs/tags/')
        run: |
          V=$(sed -n 's/^project(vmavificient VERSION \([0-9.]*\).*/\1/p' CMakeLists.txt)
          if [ "v$V" != "$GITHUB_REF_NAME" ]; then
            echo "::error::tag $GITHUB_REF_NAME does not match project VERSION $V"
            exit 1
          fi

      - uses: ./.github/actions/setup-build
        with:
          mode: vendored
          deps-path: build/deps
          deps-cache-key: cmake-deps-static-macos-arm64-${{ hashFiles('CMakeLists.txt') }}

      - name: Configure (vendored static)
        run: |
          cmake -G Ninja -B build \
            -DCMAKE_BUILD_TYPE=Release \
            -DVMAV_USE_SYSTEM_DEPS=OFF

      - name: Build
        run: cmake --build build -j$(sysctl -n hw.ncpu)

      # Tags land on commits already validated by ci.yml on main, but the
      # release binary is a different build (static, vendored) — sanity-
      # check it directly.
      - name: Run unit tests
        run: ./build/vmav_tests

      - name: Verify version banner
        run: ./build/vmavificient --help | head -1

      # SHA256SUMS lives next to the binary in the GitHub release so
      # downloaders can verify integrity with a single `shasum -c`.
      - name: Package binary + checksums
        run: |
          mkdir -p dist
          cp build/vmavificient dist/vmavificient-macos-arm64
          chmod +x dist/vmavificient-macos-arm64
          cd dist
          shasum -a 256 vmavificient-macos-arm64 > SHA256SUMS
          cat SHA256SUMS

      - name: Create GitHub release
        if: startsWith(github.ref, 'refs/tags/')
        uses: softprops/action-gh-release@3bb12739c298aeb8a4eeaf626c5b8d85266b0e65 # v2
        with:
          files: dist/*
          generate_release_notes: true
```
